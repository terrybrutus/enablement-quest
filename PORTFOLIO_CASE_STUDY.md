# Enablement Quest Portfolio Case Study

## One-Line Positioning

Enablement Quest is an interactive RPG-style case study that demonstrates performance consulting, learning architecture, sales enablement judgment, accessibility thinking, and business-impact measurement.

## Portfolio Summary

Enablement Quest: The Learning Systems Lab is a browser-based learning experience created by Terry Brutus. Players investigate realistic workplace performance problems by talking to stakeholders, reviewing evidence, diagnosing root causes, selecting enablement interventions, and earning business-impact canvases.

The experience is intentionally framed as a game, but the portfolio value is not the game mechanic by itself. The value is that the player must decide whether a workplace issue is truly a training problem or whether the better solution is workflow support, manager reinforcement, coaching, job aids, responsible AI support, or measurement design.

## Problem It Solves

Many learning and enablement portfolios show course artifacts, slide decks, or AI-generated resume tools. This project is designed to show a different capability: diagnosing the performance system before choosing the learning solution.

The core question is:

> Is this really a training problem?

That question is the throughline of the experience.

## Learning Objective

By the end of the experience, learners should be able to diagnose a workplace enablement problem, identify the root cause, select an appropriate intervention, and explain how the solution improves measurable business outcomes.

## Current Game Loop

1. A stakeholder presents a workplace problem.
2. The player gathers evidence in the environment.
3. The player identifies the root cause.
4. The player selects an enablement intervention.
5. The game shows the expected business impact.
6. The player earns a canvas summarizing the decision.

## Current Cases

### Case 01: The Broken Onboarding Portal

Scenario: New hires are taking too long to reach productivity, and leadership assumes the answer is more training.

What the player investigates:

- Stakeholder notes
- Process handoff problems
- Support ticket/ramp metrics

Intended insight:

The issue is not just a knowledge gap. The better diagnosis is unclear workflow, delayed access, and inconsistent manager reinforcement.

Earned artifact:

Enablement Diagnostic Canvas.

Business-impact story:

The proposed solution connects manager checklists, job aids, access handoff mapping, and diagnostic dashboards to faster ramp, fewer support tickets, and higher new-hire confidence.

### Case 02: The Stalled Demo Pipeline

Scenario: Sales demos are happening, but too few convert into qualified next steps.

What the player investigates:

- Demo call review
- CRM stage audit
- Manager coaching note

Intended insight:

The issue is not simply product knowledge or demo volume. The better diagnosis is shallow discovery habits and inconsistent manager coaching.

Earned artifact:

Sales Enablement Impact Canvas.

Business-impact story:

The proposed solution connects discovery guides, manager coaching rubrics, call-review practice, and pipeline inspection dashboards to improved demo-to-next-step conversion.

## What This Demonstrates To Recruiters

- Performance consulting: diagnosing before designing.
- Learning architecture: turning evidence into intervention logic.
- Sales enablement thinking: connecting enablement work to revenue behavior.
- Measurement design: tying interventions to speed, quality, adoption, productivity, and conversion signals.
- Accessibility awareness: plain language, keyboard support, readable UI, and mobile controls.
- Responsible AI framing: AI can support summarization and drafting, but humans remain accountable for judgment.
- Systems thinking: training is treated as one possible intervention, not the default answer.

## LinkedIn Post Draft

I built Enablement Quest: The Learning Systems Lab to explore a question I think every enablement and learning team should ask before building anything:

Is this really a training problem?

The experience is structured like a small RPG case study. You talk to stakeholders, inspect evidence, diagnose the root cause, choose an intervention, and earn a canvas that connects the solution to business impact.

The first case focuses on new-hire ramp. The second focuses on stalled demo conversion in a sales environment.

My goal was not to build a course about enablement terminology. It was to create a portfolio artifact that shows judgment: how to move from messy workplace signals to a practical solution that improves speed, quality, adoption, productivity, or revenue behavior.

## Current Limitations

This is still an evolving portfolio product. The current build proves the core loop and two-case structure, but the next quality bar is richer case complexity, stronger scene composition, more polished audio/transition design, and a clearer public portfolio wrapper around the playable experience.
